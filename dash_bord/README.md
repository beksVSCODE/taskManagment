# 📋 Dashboard Task Management

Корпоративный дашборд для управления задачами команды.  
**Бэкенд:** Spring Boot + PostgreSQL + JWT  
**Фронтенд:** Vue.js  
**API Base URL:** `http://localhost:8080`

---

## 🚀 Что умеет приложение

- Управлять задачами команды — создавать, редактировать, удалять
- Назначать задачи конкретным ПМ и исполнителям
- Отслеживать статус выполнения, сроки и приоритеты в Kanban-доске
- Получать аналитику по задачам и сотрудникам (графики)
- Вести коммуникацию по задачам через комментарии с @упоминаниями
- Прикреплять файлы к задачам и комментариям
- Получать уведомления о событиях (назначение, статус, просрочка)
- Управлять пользователями, отделами и проектами (для администратора)

## 🗂️ Функциональные блоки

### 🔐 Авторизация
- Вход по email + пароль → получение JWT токена
- Регистрация нового пользователя с указанием роли
- Получение информации о текущем пользователе (`/api/auth/me`)
- Токен хранится в `localStorage`, передаётся в заголовке `Authorization: Bearer <token>`

---

### 📁 Проекты
- Список всех доступных проектов (зависит от роли)
- Карточка проекта: название, описание, ПМ, отдел, статус
- Создание и редактирование проектов (ADMIN, MANAGER)
- Удаление проекта (ADMIN)

**Статусы проекта:** `ACTIVE`, `COMPLETED`, `ON_HOLD`

---

### ✅ Задачи (Kanban)
Главный экран — канбан-доска с колонками:

| Колонка | Статус |
|---|---|
| Новые | `NEW` |
| В работе | `IN_PROGRESS` |
| На проверке | `ON_REVIEW` |
| Завершено | `DONE` |

**Карточка задачи содержит:**
- Название, описание
- ПМ и исполнители
- Дата создания и срок выполнения
- Приоритет: `LOW` 🟢 / `MEDIUM` 🟡 / `HIGH` 🔴
- Теги / метки
- Индикатор просрочки 🔴 (если `isOverdue = true`)
- Счётчик подзадач (выполнено / всего)

**Фильтрация задач:**
- По исполнителю (`?assigneeId=`)
- По приоритету (`?priority=HIGH`)
- По статусу (`?status=IN_PROGRESS`)
- По проекту (`?projectId=`)

**Поиск:**
- По названию (`?title=текст`)
- По тегам (`?tagIds=1,2,3`)

**Валидация при создании:**
- Название — обязательно
- Срок выполнения — обязательно, не может быть в прошлом
- Исполнитель — минимум один

---

### 📋 Подзадачи
- Список подзадач внутри задачи
- Каждая подзадача: название, исполнитель, статус, срок
- **Автоматика:** если все подзадачи переведены в `DONE` → родительская задача тоже становится `DONE`
- Обратное: если хоть одна подзадача открыта → задача возвращается в `IN_PROGRESS`

---

### 💬 Комментарии
- Комментарии к задаче с сортировкой по времени (asc/desc)
- Поддержка @упоминаний пользователей
- Прикрепление файлов к комментарию
- Редактирование и удаление только для ADMIN

**При добавлении комментария:**
- Все участники задачи получают уведомление `NEW_COMMENT`
- Упомянутые пользователи получают отдельное уведомление `MENTION`

---

### 📎 Файлы
- Загрузка файла в два шага: сначала upload → потом привязка к задаче или комментарию
- Максимальный размер: **10 МБ**
- Допустимые форматы: jpg, png, gif, webp, pdf, doc, docx, xls, xlsx, txt, zip, rar
- Скачивание файла по кнопке
- Удаление: ADMIN или загрузивший пользователь

---

### 🔔 Уведомления

| Тип | Событие |
|---|---|
| `TASK_ASSIGNED` | Вам назначена новая задача |
| `STATUS_CHANGED` | Изменился статус задачи |
| `NEW_COMMENT` | Новый комментарий к вашей задаче |
| `MENTION` | Вас упомянули в комментарии |
| `TASK_OVERDUE` | Задача просрочена (каждый день в 09:00) |

- Счётчик непрочитанных (`/api/notifications/count`) — для бейджа на колокольчике
- При открытии задачи все уведомления по ней автоматически помечаются прочитанными
- Кнопка "Прочитать все"

---

### 📊 Аналитика (только ADMIN и MANAGER)

| Эндпоинт | Что показывает |
|---|---|
| `/api/analytics/tasks/by-status` | Кол-во задач по каждому статусу |
| `/api/analytics/tasks/by-priority` | Кол-во задач по приоритетам |
| `/api/analytics/employees/performance` | Производительность каждого сотрудника |
| `/api/analytics/tasks/by-period` | График задач по дням/неделям/месяцам |
| `/api/analytics/tasks/completion-ratio` | Соотношение завершённых и просроченных |
| `/api/analytics/tasks/{id}/history` | История изменений задачи |

**Производительность сотрудника включает:**
- Всего назначено задач
- Выполнено (статус DONE)
- Просрочено (dueDate прошёл, статус не DONE)
- В работе (статус IN_PROGRESS)
- Процент выполнения

**График по периодам (`by-period`):**
```
GET /api/analytics/tasks/by-period?groupBy=month&from=2024-01-01&to=2024-12-31
```
Параметр `groupBy`: `day` | `week` | `month`

## 🛠️ Настройка Axios (Vue.js)

```js
// src/api/axios.js
import axios from 'axios'
import router from '@/router'

const api = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 10000
})

// Автоматически добавляем токен к каждому запросу
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// Если токен протух — редирект на /login
api.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      router.push('/login')
    }
    return Promise.reject(err)
  }
)

export default api
```

## 📋 Коды ответов API

| Код | Значение |
|---|---|
| `200` | Успех |
| `204` | Успех, нет тела ответа (DELETE) |
| `400` | Ошибка валидации |
| `401` | Не авторизован (нет или истёк токен) |
| `403` | Нет прав доступа для данной роли |
| `404` | Ресурс не найден |
| `500` | Ошибка сервера |
